# Open Questions — founding questions kept deliberately unanswered

**Status:** Open questions (founding, unresolved) · **Date:** 2026-06-27
**Not ideas. Not answers. Not for the manifesto.**

The incubating framings hold *answers* that haven't earned canon yet. This document holds something
different: **questions** that have no answer and *deserve to stay questions for now.* Keeping certain
things as questions — rather than rushing to answer them — preserves the project's ability to
**discover**, not merely to confirm its vision. It is the mirror of the promotion rule: as important
*not* to canonize prematurely as it is to canonize only what is proven.

> **A question is an asset, not a gap.** In most projects an open question is felt as something
> missing. Here it is an object we protect until reality lets us answer it.

## Theme 1 — Intent / will (the dimension the manifesto barely touches)

The manifesto describes knowledge thoroughly and *will* almost not at all. Yet a project is not only a
body of knowledge: it pursues a goal, changes strategy, arbitrates, makes compromises, abandons
paths. It has **intention**. Today the chain is:

```
Knowledge → Decision → Project evolution
```

It says almost nothing about:

```
Intent → Knowledge → Decision → Evolution
```

Not a layer to add today. But if Daryl truly becomes a project's cognitive system, it may eventually
need to govern not only what a project *knows* but what it *seeks to accomplish*. Open questions:

- What *is* project intent?
- Can intent be governed (the way claims are)?
- Is a **Goal** a Knowledge Object — or something else?
- Are **constraints** first-class citizens?
- Can a project legitimately *disagree with its own past intent*?

## Theme 2 — Can definitions evolve under governance?

A definition — say *Knowledge Object* — is treated today as fixed. In five years it will likely have
evolved. What if a definition is itself a **Knowledge Object**: with its own Acts, supersessions,
agent consultations, standing, and history? Then the *words themselves become governed.* It is a
natural consequence of the system: if a project accumulates governed knowledge, its **vocabulary**
must become governable too.

- Can a definition carry a governed lifecycle (Acts, supersession, standing)?
- Is the Vocabulary (ADR-PRL-0003) a *set of Knowledge Objects* rather than a frozen glossary?
- Who ratifies a definitional supersession — and against what evidence?
- Can two definitions of the same term coexist with different standing during a transition?

Not a layer today; kept as questions.

## How to use this file

These stay questions until either an experiment or a real need forces an answer — at which point the
answer goes through the normal path (an ADR for law, the manifesto for vision, a framing for
positioning). Adding a question here costs nothing and protects optionality. Answering one
prematurely would cost a great deal.
